# YT Level — YouTube भाषा स्तर विश्लेषक

**स्थानीय AI का उपयोग करके किसी भी YouTube वीडियो के CEFR स्तर (A1–C2) का विश्लेषण करें — कोई API कुंजी नहीं, कोई इंटरनेट आवश्यक नहीं।**

दो AI इंजनों में से चुनें: **Gemini Nano** (Chrome में निर्मित) या **Ollama** (स्थानीय सर्वर)। **किसी भी भाषा** (अंग्रेज़ी, स्पेनिश, फ्रेंच, जर्मन, चीनी, आदि) के लिए काम करता है। एक्सटेंशन वीडियो ट्रांसक्रिप्ट लाकर उसका CEFR स्तर वर्गीकृत करता है। प्रत्येक वीडियो थंबनेल पर एक रंगीन बैज दिखाई देता है।

## विशेषताएँ

- 🏷️ **CEFR बैज** — YouTube वीडियो थंबनेल पर रंगीन वृत्त (A1–C2)
- 🤖 **दो AI इंजन** — **Gemini Nano** (बिल्ट-इन Chrome AI) या **Ollama** (स्थानीय मॉडल) का उपयोग करें
- 🌍 **बहु-भाषा** — किसी भी भाषा के वीडियो का विश्लेषण करता है
- 🎨 **कस्टम Ollama सर्वर** — अपने नेटवर्क पर किसी भी Ollama इंस्टेंस से जुड़ें
- ⚡ **त्वरित कैश** — परिणाम स्थानीय रूप से कैश किए जाते हैं, पुनर्विश्लेषण से बचाता है
- 🔒 **100% निजी** — सब कुछ स्थानीय रूप से चलता है, कोई डेटा आपकी मशीन से बाहर नहीं जाता

## आवश्यकताएँ

- **Chrome 128+**, **Brave**, या कोई भी Chromium-आधारित ब्राउज़र
- **Gemini Nano**: Chrome 128+ जिसमें Prompt API सक्षम हो (नीचे देखें)
- **Ollama**: Ollama स्थापित और चालू ([ollama.com](https://ollama.com)) और कम से कम एक मॉडल डाउनलोड हो

## इंस्टॉलेशन — Gemini Nano

Gemini Nano Chrome का बिल्ट-इन AI मॉडल है। किसी डाउनलोड या सर्वर की आवश्यकता नहीं।

### 1. Prompt API फ़्लैग सक्षम करें

1. **`chrome://flags/#prompt-api-for-gemini-nano`** खोलें
2. फ़्लैग को **"Enabled"** पर सेट करें
3. Chrome को पुनरारंभ करने के लिए **"Relaunch"** पर क्लिक करें

### 2. एक्सटेंशन में मॉडल स्थिति जाँचें

1. YT Level एक्सटेंशन आइकन पर क्लिक करें
2. **Gemini Nano** टैब चुनें
3. स्थिति दिखाएगी:
   - **Available** — उपयोग के लिए तैयार
   - **Downloading** — मॉडल डाउनलोड हो रहा है
   - **Downloadable** — पहले डाउनलोड करना होगा (डाउनलोड ट्रिगर करने के लिए क्लिक करें)
   - **Unavailable** — आपके ब्राउज़र में समर्थित नहीं

### 3. विश्लेषण भाषा चुनें

Gemini Nano टैब में, उस वीडियो की भाषा चुनें जिसका आप विश्लेषण करना चाहते हैं:

| कोड | भाषा   |
|-----|--------|
| en  | English|
| es  | Spanish|
| ja  | Japanese|
| de  | German |
| fr  | French |

### 4. प्रयास मोड चुनें

- **Quick** — सरल प्रॉम्प्ट के साथ तेज़ वर्गीकरण
- **Deep** — व्यापक प्रॉम्प्ट के साथ विस्तृत CEFR मूल्यांकन

### 5. एक्सटेंशन लोड करें

1. **`chrome://extensions`** (या **`brave://extensions`**) पर जाएँ
2. **"डेवलपर मोड"** चालू करें (ऊपरी दाएँ कोने में)
3. **"लोड अनपैक्ड"** पर क्लिक करें
4. प्रोजेक्ट फ़ोल्डर चुनें

### 6. अनुमतियाँ दें (महत्वपूर्ण)

1. `chrome://extensions` में **YT Level** पर **"विवरण"** क्लिक करें
2. **"इस एक्सटेंशन को आपके द्वारा देखी जाने वाली वेबसाइटों पर सभी डेटा पढ़ने और बदलने की अनुमति दें"** चालू करें
3. यदि संकेत दिखे, तो **"अनुमति दें"** पर क्लिक करें

> इस चरण के बिना, एक्सटेंशन लोड तो होता है लेकिन YouTube पेजों पर नहीं चलेगा।

## इंस्टॉलेशन — Ollama

### 1. Ollama स्थापित करें

**Linux / macOS:**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

**Windows:**
[ollama.com/download](https://ollama.com/download) से इंस्टॉलर डाउनलोड करें और चलाएँ। Ollama स्वचालित रूप से बैकग्राउंड सेवा के रूप में शुरू हो जाएगा।

### 2. एक मॉडल डाउनलोड करें

```bash
ollama pull gemma3:1b
```

> आप किसी भी मॉडल का उपयोग कर सकते हैं। एक्सटेंशन आपको पॉपअप से चुनने देता है कि कौन सा मॉडल उपयोग करना है।

### 3. Ollama में CORS कॉन्फ़िगर करें

एक्सटेंशन को YouTube वेबसाइट से Ollama से बात करने की अनुमति चाहिए।

#### Linux — विकल्प A: Systemd (स्थायी, अनुशंसित)

```bash
sudo mkdir -p /etc/systemd/system/ollama.service.d

echo '[Service]
Environment=OLLAMA_ORIGINS=*' | sudo tee /etc/systemd/system/ollama.service.d/override.conf

sudo systemctl daemon-reload
sudo systemctl restart ollama
```

#### Linux — विकल्प B: मैन्युअल (अस्थायी)

```bash
sudo systemctl stop ollama
OLLAMA_ORIGINS=* ollama serve
```

#### Windows — विकल्प A: स्थायी (अनुशंसित)

1. **सिस्टम गुण** → **पर्यावरण चर** खोलें
2. एक नया **सिस्टम चर** जोड़ें:
   - नाम: `OLLAMA_ORIGINS`
   - मान: `*`
3. **OK** पर क्लिक करें और सिस्टम ट्रे से Ollama को पुनरारंभ करें (राइट-क्लिक → बंद करें, फिर पुनः प्रारंभ करें)

#### Windows — विकल्प B: अस्थायी (PowerShell)

```powershell
$env:OLLAMA_ORIGINS="*"
ollama serve
```

> Windows पर, सिस्टम ट्रे से Ollama बंद करने के बाद ये कमांड चलाएँ।

### 4. एक्सटेंशन लोड करें

ऊपर Gemini Nano अनुभाग में चरण 5 और 6 के समान।

### 5. Ollama के साथ एक्सटेंशन का उपयोग करें

1. एक्सटेंशन आइकन पर क्लिक करें
2. **Ollama** टैब चुनें
3. अपना सर्वर URL सेट करें (डिफ़ॉल्ट: `http://localhost:11434`)
4. कनेक्शन का परीक्षण करने के लिए **OK** पर क्लिक करें
5. ड्रॉपडाउन से एक मॉडल चुनें

## एक्सटेंशन का उपयोग करना

1. **https://www.youtube.com** पर जाएँ
2. ट्रांसक्रिप्ट वाले वीडियो विश्लेषण के दौरान हरा स्पिनर दिखाते हैं
3. स्तर के साथ एक रंगीन वृत्त दिखाई देता है: **A1**, **A2**, **B1**, **B2**, **C1**, या **C2**
4. बैज पर होवर करें यह देखने के लिए कि कौन सा इंजन और मॉडल उपयोग किया गया
5. पॉपअप खोलने और इंजनों के बीच स्विच करने के लिए एक्सटेंशन आइकन पर क्लिक करें

## यह कैसे काम करता है

1. YouTube फ़ीड से प्रत्येक वीडियो ID निकालता है
2. `youtube-transcript.ai` के माध्यम से ट्रांसक्रिप्ट प्राप्त करता है
3. CEFR वर्गीकरण के लिए ट्रांसक्रिप्ट चयनित AI इंजन (Gemini Nano या Ollama) को भेजता है
4. वीडियो थंबनेल पर परिणाम को वृत्ताकार बैज के रूप में प्रदर्शित करता है
5. परिणाम स्थानीय रूप से कैश किए जाते हैं ताकि पुनर्विश्लेषण न हो

## कस्टम Ollama सर्वर

डिफ़ॉल्ट रूप से एक्सटेंशन `http://localhost:11434` से जुड़ता है। आप इसे बदल सकते हैं:

1. एक्सटेंशन आइकन पर क्लिक करें
2. **Ollama** टैब चुनें
3. अपना सर्वर URL दर्ज करें (जैसे `http://192.168.1.100:11434`)
4. **OK** पर क्लिक करें — एक्सटेंशन कनेक्शन का परीक्षण करेगा और उपलब्ध मॉडल लोड करेगा
5. डिफ़ॉल्ट पर रीसेट करने के लिए **↺** पर क्लिक करें

## नोट्स

- केवल उन वीडियो का विश्लेषण करता है जिनमें YouTube पर **ट्रांसक्रिप्ट उपलब्ध** है
- विश्लेषण का समय आपके हार्डवेयर पर निर्भर करता है (Gemini Nano के साथ तेज़, CPU पर Ollama के साथ प्रति वीडियो 20–60 सेकंड)
- यदि कोई इंजन उपलब्ध नहीं है, तो कोई बैज नहीं दिखाया जाएगा
- किसी API कुंजी या इंटरनेट कनेक्शन की आवश्यकता नहीं
- सभी डेटा स्थानीय रहता है — कुछ भी बाहरी सर्वर को नहीं भेजा जाता
